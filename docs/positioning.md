## gh-verify の独自性

1. **形式検証されたルールロジック** — Creusot + SMTで判定ロジック自体の正しさを数学的に証明。競合にこの層はない
2. **lint的パラダイム** — 大量のルールを決定的・ヒューリスティックに高速実行
3. **仕様マッピング** — PRが事前定義された仕様とマッピング可能かを検知し変更の妥当性を検証
4. **SOCレポートのカスタマイズ生成** — 組織運用に応じたコンプライアンス文書を検査結果から生成 (構想段階)
5. **GitHub外ツール連携** — Jira等との連携を設計に内包

## 競合との軸の違い

| 軸 | gh-verify | 競合 |
|----|-----------|------|
| 正しさの保証対象 | ルールロジック自体 (形式検証) | データの改竄防止 (暗号署名) / プロセスの強制 |
| 検査スタイル | lint的: 大量ルール高速実行 | ワークフロー自動化 / 証跡収集 |
| カバー範囲 | 設計〜リリースの開発プロセス全体 | PRマージ〜デプロイ中心 |
| 外部ツール連携 | 設計に内包 (Jira等) | GitHub中心 / パートナーエコシステム |

## 不足しているもの

| 機能 | 対応状況 | 優先度 |
|------|----------|--------|
| GitHub外ツール連携 (Jira等) | 設計に余地あり・未実装 | 高 |
| SOCレポート生成 | 構想段階 | 高 |
| ダッシュボード / Web UI | なし | 中 |
| パートナーエコシステム | なし | 低 |
